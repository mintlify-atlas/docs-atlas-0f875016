## What is Microservicios Ecosystem?

Microservicios Ecosystem is a fully-featured reference implementation of a microservices architecture built on **Spring Boot 3.4** and **Java 21**. It demonstrates industry best practices including hexagonal architecture, event-driven design, distributed tracing, and centralized observability — all deployable with a single Docker Compose command.

## Services at a glance

| Service | Port | Role |
|---|---|---|
| **config-server** | 8888 | Centralized configuration (filesystem) |
| **eureka-server** | 8761 | Service discovery registry |
| **auth-service** | 8083 | JWT authentication + BCrypt |
| **api-gateway** | 8080 | Routing + OAuth2 token validation |
| **inventory-service** | 8082 | Product CRUD (hexagonal architecture) |
| **notification-service** | 8084 | Async event consumer |
| **RabbitMQ** | 5672 / 15672 | Message broker (option 1) |
| **Apache Kafka** | 9092 | Message broker (option 2) |
| **Zipkin** | 9411 | Distributed tracing UI |
| **Elasticsearch** | 9200 | Log storage |
| **Logstash** | 5044 | Log ingestion + parsing |
| **Kibana** | 5601 | Log visualization UI |

## How requests flow

```
Client
  └─► API Gateway (8080)          — validates JWT, routes request
        └─► inventory-service (8082) — hexagonal domain logic, persists to H2
              └─► Message Broker     — publishes ProductCreatedEvent
                    └─► notification-service (8084) — logs notification
```

All services register with Eureka at startup, pull configuration from Config Server, and report traces to Zipkin. Logs flow over TCP to Logstash → Elasticsearch → Kibana.
